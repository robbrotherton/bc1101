Correlation and Regression

Part 1: Computing and graphing associations

Instructions

Correlation

R has two basic functions for calculating correlations: cor(), and cor.test() . They each require two arguments, named x and y, referring to the two variables (i.e. columns of data) we want to compute the correlation between.

# first read in the relevant data; for this example it's a spreadsheet with two
# columns, named x and y
df <- read.csv("data/10_example.csv")

cor(df$x, df$y)
[1] -0.09394529
cor.test(df$x, df$y)

As you will see, cor() simply returns the value of Pearson‚Äôs \(r\). cor.test() returns all the information you need to complete a significance test, i.e. the correlation coefficient, degrees of freedom, and \(p\)-value as well as some stuff you can ignore for our purposes, such as \(t\) and a confidence interval.

Regression

The lm() function computes a ‚Äòlinear model‚Äô, i.e. the kind of linear regression model that we have learned about in the lectures. Rather than accepting x and y arguments, the regression function requires a formula, just like you‚Äôve used with t.test() and aov() in previous problem sets. The idea is the same here: we articulate a formula in the form DV ~ IV. In this context, the DV is the outcome variable, i.e. the one we want to predict. The IV is the predictor. And remember, I‚Äôm using DV and IV generically here: when you write your code you will replace them with the names of columns in your data.frame.

There is a second step though. Since lm() doesn‚Äôt give us all the information about the regression model that we usually need to report, we assign the model (the output of the lm() function) to a name (model, in my case), and then we feed that into the summary() function.

model <- lm(y ~ x, data = df)

summary(model)

Visualizing correlation & regression

To visualize bivariate correlations, you can use ggplot, specifying your x and y variables as aesthetics, and adding a geom_point() layer for the dots.

library(ggplot2)

ggplot(df, aes(x = x, y = y)) +
  geom_point()

To make this a visualization of the regression model as well, we can simply add another layer to show the best fit line. This is computed by geom_smooth. We just need to tell it the method we want it to use to compute the line. For our purposes, method = "lm" meaning ‚Äúlinear model‚Äù.

ggplot(df, aes(x = x, y = y)) +
  geom_point() +
  geom_smooth(method = "lm")
`geom_smooth()` using formula = 'y ~ x'

Part 2. The World Happiness Report.

Instructions

For this question you will work with some real data from the World Happiness Report.

Look at the data after you read it into R to get an idea of what variables it contains. The column names generally give a clear indication of what the scores are, but some may be unclear or unfamiliar. There‚Äôs a full explanation at in the WHR appendix here.

You‚Äôll be using cor() and/or cor.test() to examine correlations. The only new thing to be aware of is that, if you try to correlate two variables and there is even a single missing data point (indicated by NA), you will get an answer of NA. To avoid this, you can specify use = "pairwise.complete.obs as an argument within either function. That will cause R to ignore cases where one data point is missing, and use however many have complete data for both variables.
